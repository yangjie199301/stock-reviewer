# 基本面分析 — 计划

> 本文档属于 `plans/analysis/` 目录，分析与策略相关 plan 放这里。

---

## 1. 目标

从基本面角度评估个股质地，为选股和复盘提供参考依据。

---

## 2. 功能拆解

### 2.1 财务数据获取

| 数据     | 说明                                        | 数据源                       |
| -------- | ------------------------------------------- | ---------------------------- |
| 三大报表 | 资产负债表、利润表、现金流量表（季度/年度） | akshare `financial_report_*` |
| 历史财务 | 过去 5 年的关键财务指标                     | akshare                      |
| 行业归属 | 所属行业、同行业公司列表                    | akshare / tushare            |

### 2.2 核心指标计算

| 指标                | 计算公式 / 来源                  | 用途     |
| ------------------- | -------------------------------- | -------- |
| PE（市盈率）        | 股价 / 每股收益                  | 估值     |
| PB（市净率）        | 股价 / 每股净资产                | 估值     |
| ROE（净资产收益率） | 净利润 / 净资产                  | 盈利质量 |
| 毛利率              | (营收 - 营业成本) / 营收         | 盈利能力 |
| 营收增长率          | (本期营收 - 上期营收) / 上期营收 | 成长性   |
| 净利润增长率        | (本期净利 - 上期净利) / 上期净利 | 成长性   |
| 资产负债率          | 总负债 / 总资产                  | 偿债风险 |

### 2.3 分析展示

- 单只股票基本面概览卡片（关键指标一览）
- 同行业横向对比（选出行业内排名）
- 历史趋势（过去 N 年 ROE/营收趋势图）

### 2.5 数据更新频次

不同数据的变化频率差异很大，按频次分层处理：

| 数据                          | 变化频率                           | 更新策略                       |
| ----------------------------- | ---------------------------------- | ------------------------------ |
| **三大报表**                  | 每季度一次（A 股固定披露期见下表） | 季度末触发批量更新             |
| **历史财务数据**（过去 5 年） | 基本不变                           | 仅首次拉取时写入，后续不再刷新 |
| **行业归属**                  | 几乎不变（除非公司重组）           | 首次拉取后按需手动刷新         |
| **PE / PB**                   | 每交易日随股价变化                 | 跟随日线行情每天更新           |
| **ROE / 毛利率**              | 随财报更新（约季度级）             | 财报更新后重新计算             |
| **营收/利润增长率**           | 随财报更新（约季度级）             | 财报更新后重新计算             |

**A 股财报披露时间表：**

| 报告           | 对应期间    | 最晚披露时间 |
| -------------- | ----------- | ------------ |
| 年报           | 1/1 ~ 12/31 | 次年 4/30    |
| 一季报         | 1/1 ~ 3/31  | 当年 4/30    |
| 中报（半年报） | 1/1 ~ 6/30  | 当年 8/31    |
| 三季报         | 1/1 ~ 9/30  | 当年 10/31   |

> 密集披露期集中在 4 月（年报+一季报同时出）、8 月（中报）、10 月（三季报）。
> 数据库里加一列 `report_date` 记录实际披露日，用于判断数据是否已更新。

### 2.6 首次与增量拉取策略

| 场景       | 操作                                                                     |
| ---------- | ------------------------------------------------------------------------ |
| 第一次运行 | 批量拉取全部股票池的近 5 年财报 + 当前最新季度                           |
| 日常运行   | 只检查最新季度是否已披露（当前日期是否在该季度的披露期内），已披露则拉取 |
| 手动触发   | 支持指定股票 + 报告期强制刷新                                            |

### 2.4 基本面评分

> 设计目标：简单、直观、可解释。不追求复杂的模型，让用户一眼看懂这家公司基本面"好不好"。

#### 评分维度

每个维度 0-10 分，加权汇总为总分（满分 100）。

| 维度           | 权重 | 评价内容                 | 打分逻辑                                              |
| -------------- | ---- | ------------------------ | ----------------------------------------------------- |
| **盈利能力**   | 30%  | ROE、毛利率              | 近 3 年平均 ROE > 15% 且毛利率 > 30% → 高分           |
| **成长性**     | 25%  | 营收增长率、净利润增长率 | 过去 3 年营收 CAGR > 10% 且净利增速 > 营收增速 → 高分 |
| **偿债安全**   | 15%  | 资产负债率、流动比率     | 负债率 30%-60% 且流动比率 > 1.5 → 高分                |
| **估值合理**   | 15%  | PE、PB 在同行业的百分位  | PE/PB 处于行业中低分位 → 高分（不在极端高位）         |
| **现金流质量** | 15%  | 经营现金流 / 净利润      | 比率 > 1（赚的钱都收了回来）→ 高分                    |

#### 打分示例（工业富联为例，仅示意逻辑）

```
盈利能力  8/10  × 30% = 2.4
成长性    7/10  × 25% = 1.75
偿债安全  6/10  × 15% = 0.9
估值合理  7/10  × 15% = 1.05
现金流质量 8/10 × 15% = 1.2
─────────────────────
总分              = 73/100   质地评级：良好 ✅
```

#### 质地评级

| 总分范围 | 评级 | 标记 |
| -------- | ---- | ---- |
| ≥ 85     | 优秀 | 🟢   |
| 70 ~ 84  | 良好 | ✅   |
| 50 ~ 69  | 一般 | ⚠️   |
| < 50     | 较差 | 🔴   |

#### 评分使用场景

- 复盘文档中标注持仓/关注股票的评级和总分
- 评分大幅变化（如从 70 跌到 50）时自动提示
- 支持按评分筛选股票池

> **注意：** 评分目的是辅助判断，不是交易信号。权重和阈值后期可根据用户的交易风格调整。

## 3. 已确认的决定

- **数据存储策略：** 拉取后存在本地 SQLite，减少请求次数
- **自定义指标：** 不支持，使用固定指标体系
- **同行业对比范围：** 只跟行业前 10 名比（按营收规模排序）
- **重大变化阈值：** 营收增长 > 50%（积极）或 < -30%（预警）；净利润同理
- **评分权重：** 不支持手动调整，使用固定权重
