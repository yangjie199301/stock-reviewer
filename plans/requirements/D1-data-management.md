# D1 — 数据管理

> 需求总览索引：[requirements.md](requirements.md)
> 前置依赖：无（这是最底层的基础设施）
> 为 D2 每日复盘提供数据支撑

---

## 1. 用户故事

系统需要能够从多个数据源获取原始数据，统一存储到本地，支持按需查询和整合，为上层所有分析和复盘功能提供数据基础。

**用户原话：**

> "在每日复盘之前需要有一个数据获取、存储、查询、整合"

---

## 2. 功能拆解

### 2.1 数据获取

**支持两种数据获取模式：**

| 模式 | 状态 | 说明 |
|------|------|------|
| **通达信导入** | ✅ **当前主要方式** | 从通达信导出的 CSV 文件解析入库，见 `infrastructure/adapters/tdx_importer.py` |
| **在线拉取** | ⏳ 待实现 | 通过 akshare 等公开接口在线获取，接口签名已预留，内部抛出 NotImplementedError |

#### 在线拉取（规划中）

| 数据类别 | 具体内容 | 数据源 | 更新频次 | 具体接口 |
| -------- | -------- | ------ | -------- | -------- |
| **日线行情** | 开高低收量、涨幅、换手率 | akshare | 每日（增量） | `stock_zh_a_hist()` |
| **龙虎榜** | 上榜个股、净买卖、上榜原因、机构买卖 | akshare | 每日 | `stock_lhb_detail_em()`、`stock_lhb_jgmmtj_em()` |
| **资金流向** | 个股主力净流入/出 | akshare | 每日 | `stock_individual_fund_flow()` |
| **北向资金** | 沪/深股通净流入 | akshare | 每日 | `stock_hsgt_summary_em()` |
| **行业板块** | 申万行业 / 概念板块涨跌 | akshare | 每日 | `stock_board_industry_name_em()`、`stock_board_concept_name_em()` |
| **政策资讯** | 国务院政策文件 | 爬虫 | 每日 | 待定 |
| **研报** | 券商研报标题/摘要 | 爬虫 | 每日 | 待定 |
| **基本面** | 三大报表、财务指标 | akshare | 季度 | `financial_report_*` |

#### 通达信导入（当前方式）

- **方式一（推荐）：** 下载官方日线数据完整包
  - 下载链接：`https://data.tdx.com.cn/vipdoc/hsjday.zip`
  - 解压后目录结构：`hsjday/sh/lday/*.day`、`hsjday/sz/lday/*.day`
  - 解析 `.day` 二进制文件（32 字节/记录，含开高低收量额）
  - 支持自动下载：`download_latest_data()`
- **方式二：** 用户从通达信手动导出日线 CSV 文件
  - 将文件放入 `data/imports/` 目录
  - 支持批量导出格式（含代码、名称、日期、开高低收量额等列）
  - 支持单个文件（不含代码列，通过文件名识别）
  - 支持 UTF-8 / GBK 编码，自动识别表头列名
- 导入时自动去重，增量写入

### 2.2 数据存储

- 使用 SQLite，单文件存储，无需部署
- 每类数据独立表，表结构见附录
- 自动去重（按日期+代码唯一键）
- 增量写入，不覆盖历史

### 2.3 数据查询

| 查询类型     | 说明                            | 使用场景       |
| ------------ | ------------------------------- | -------------- |
| **按日期查** | 查某一天或某区间的全部/某类数据 | 每日复盘       |
| **按股票查** | 查某只股票的所有关联数据        | 个股分析       |
| **最新状态** | 查每只股票最新一天的行情        | 行情快照       |
| **缺失检查** | 检查哪些日期/股票缺少数据       | 数据完整性校验 |

### 2.4 数据整合

- 将分表数据按需组装为"一份完整的当日数据"供给复盘使用
- 整合维度：大盘 + 板块 + 个股 + 资金 + 情绪
- 部分数据缺失时，该维度标记"暂无数据"，不影响整体

---

## 3. 已确认的决定

- **股票池范围：** 全市场（全部 A 股）
- **爬虫策略：** 模拟浏览器访问（设置 User-Agent、Referer、Cookie 等请求头，不使用代理），适当加入随机延时控制访问频率
- **SQLite 路径：** 支持指定路径（可通过配置文件或环境变量指定数据库文件位置）
